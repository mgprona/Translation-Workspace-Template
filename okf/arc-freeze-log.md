# OKF Arc Freeze Log

บันทึกการ **freeze ศัพท์ระดับ arc** — ทำก่อนเข้า Phase B (Edit) ของแต่ละ arc เสมอ

เหตุผล: ตอนจบ Phase A (draft ทั้ง arc ครบ) OKF "เห็นเรื่องทั้ง arc แล้ว" จึงล็อกศัพท์ของ arc นั้นให้นิ่ง แล้วเกลาทั้ง arc ด้วยมาตรฐานเดียว — กัน drift ที่เกิดจากการเกลาตอน OKF ยังไม่นิ่ง

## Freeze แบบอ่อน (soft-freeze)

ศัพท์ที่ freeze แล้วถือเป็นมาตรฐาน **แต่แก้ย้อนได้ถ้ามีเหตุผล** (เช่น arc หลังพบว่าคำเดิมแปลพลาด) โดยต้อง:

1. บันทึกการแก้ในตาราง "การแก้ย้อน" ด้านล่าง
2. แก้ OKF ไฟล์ที่เกี่ยวข้อง
3. รัน `etc/replace-term.ps1` ไล่แก้ทุกตอนใน arc ที่ได้รับผลกระทบ (encoding-safe — ห้าม find-replace ด้วยมือหรือสคริปต์อื่น)
4. รัน `etc/check-encoding.ps1` ยืนยันไฟล์ไม่พังหลังแก้

> ความเสี่ยงที่ยอมรับ: ถ้าแก้ย้อนศัพท์ของ arc ที่ **ส่งมอบไปแล้ว** ไฟล์ที่เผยแพร่จะไม่ตรงกับฉบับแก้ — ชั่งน้ำหนักก่อนแก้
