# Prompt: Polish Chapter (Phase B — เกลาทั้ง arc)

เกลาสำนวนบทแปลตอนที่ `{CHAPTER_NUMBER}` ให้เป็นนิยายไทยระดับมืออาชีพ

> **บริบท arc model**: prompt นี้คือ **Phase B** ทำ *หลัง* draft+QA ครบทั้ง arc แล้ว (gate A→B ผ่าน + OKF freeze แล้ว) — เกลาทั้ง arc รวดทีละตอนด้วย OKF ที่นิ่งแล้ว ไม่ใช่เกลาแทรกระหว่าง draft

## Files

ไฟล์ร่าง:

`thai_draft/ch{CHAPTER_NUMBER}.md`

รายงาน QA:

`qa/reports/ch{CHAPTER_NUMBER}-qa.md`

OKF:

`okf/`

## ⛔ Gate ก่อนเริ่ม (บังคับ)

**ก่อนเริ่ม Phase B ของ arc** (ทำครั้งเดียวตอนเปิดเฟส) — ตรวจว่าทั้ง arc พร้อม:

```powershell
powershell -File etc/check-arc-phase.ps1 -Arc {ARC_NUMBER} -Phase B
```

ถ้า `[FAIL]` → ยังมีตอนค้าง QA หรือยังไม่ freeze OKF → **ห้ามเริ่มเกลา** กลับไปทำ Phase A ให้ครบก่อน

**รายตอน** ก่อนเกลาแต่ละตอน ต้องรัน verify-chapter ยืนยันว่าร่างแปลและรายงาน QA มีอยู่จริง:

```powershell
powershell -File etc/verify-chapter.ps1 -Chapter {CHAPTER_NUMBER} -Stage draft
powershell -File etc/verify-chapter.ps1 -Chapter {CHAPTER_NUMBER} -Stage qa
```

- ทั้งสองคำสั่งต้อง exit 0 จึงเกลาต่อได้
- ถ้าอันใด `[FAIL]` → **ห้ามสร้าง `thai_edited/`, ห้ามแตะ chapter-status.md** ให้รายงานว่าตอนนี้ยังไม่ผ่านขั้น draft/QA จริง แล้วหยุด

> **ถ้าต้องเปลี่ยนคำมาตรฐานระหว่างเกลา** (OKF freeze แบบอ่อน): ห้าม find-replace ด้วยมือหรือสคริปต์อื่น ให้ใช้ `etc/replace-term.ps1 -Old ... -New ... -Arc {ARC_NUMBER}` (DryRun ก่อน แล้ว `-Apply`) + บันทึกใน `okf/arc-freeze-log.md`

## ก่อนเริ่ม

อ่าน **ปรัชญาการแปล** และ genre profile ใน `okf/style-guide.md` ก่อน — เป้าคือทำให้**อ่านลื่นบนมือถือ คงบรรยากาศเดิม ไม่ทำให้หรูขึ้น ไม่เพิ่มของ**

## 4 มิติการเกลา (ทั้งหมดมุ่งสู่ "ลื่น" ไม่ใช่ "หรู")

### มิติ 1: ลบ Translation-ese

ปรับโครงสร้างประโยคที่ติดมาจากภาษาต้นทาง:

- ลด ที่/ซึ่ง/ของ/ได้/แล้ว/อยู่ ที่ซ้ำซ้อนหรือไม่จำเป็น
- เปลี่ยน passive voice ที่ไม่เป็นธรรมชาติในไทย เช่น "เขาถูกมองด้วยสายตา..." → "สายตา...จับจ้องมาที่เขา"
- ปรับลำดับคำให้เป็นจังหวะไทย ไม่ใช่ลำดับ subject-verb-object แบบอังกฤษทุกประโยค
- ลบคำเชื่อมที่ไม่จำเป็น เช่น "และ" "แต่" "อย่างไรก็ตาม" ที่ขึ้นต้นประโยคบ่อยเกินไป
- **ละประธานให้มากที่สุด** — ภาษาไทยละประธานได้บ่อยกว่าอังกฤษมาก นี่คือจุดที่ translation-ese หลงเหลือบ่อยสุด:
  - "เขา/เธอ/ข้า/มัน" ที่ขึ้นต้นประโยคติดๆ กัน → ละทิ้งเมื่อบริบทชัด หรือสลับใช้ชื่อตัวละคร
  - โดยเฉพาะเรื่องมุมมองบุรุษที่ 1 ("ข้า...ข้า...ข้า" ทุกประโยค) → ละ "ข้า" ออกเมื่อประธานชัดอยู่แล้ว เช่น "ข้ายกมือขึ้น ข้าเอ่ยเสียงดัง" → "ยกมือขึ้น เอ่ยเสียงดัง"
  - อ่านออกเสียง — ถ้าได้ยิน "ข้า/เขา" ถี่เกินจริง แสดงว่าต้องตัด

### มิติ 2: เลือกคำให้ลื่นและเข้าแนว (Word Choice — ลื่น ไม่ใช่หรู)

> **เป้าหมายคือ "เห็นปุ๊บเข้าใจปั๊บ" ไม่ใช่ "อลังการ"** — ดูปรัชญาการแปลใน `style-guide.md`

- ตรวจคำนาม/กริยา/คุณศัพท์สำคัญ — มีคำไทยที่**ลื่นกว่า อ่านง่ายกว่า**และยังเข้าแนวไหม?
- **ห้ามยกระดับคำให้หรูเกินต้นฉบับ** เพราะทำให้อ่านเหนื่อย เช่น อย่าเปลี่ยน "ฝ่าศึกมานับไม่ถ้วน" → "ฝ่าฟันขุนเขาดาบป่ากระบี่อันโชกโชน" (หรูจนสะดุด)
- เลี่ยงคำประสมยืดยาว/สำนวนโบราณที่ต้องตีความ — เลือกคำที่คนไทยใช้จริง
- **คงศัพท์เฉพาะของแนว** (ปราณ, วิชา, สำนัก, ชื่อตาม OKF) — เกลาแค่คำบรรยายทั่วไปรอบๆ ให้ลื่น
- หลีกเลี่ยงคำที่ `style-guide.md` ระบุว่าต้องหลีก

### มิติ 3: ขัดคำบรรยายให้คม (Description — คม ไม่ใช่เพิ่ม)

ทำให้คำบรรยายที่มีอยู่**คมและลื่นขึ้น โดยไม่เพิ่มของใหม่**:

- ตัดคำฟุ่มเฟือย/คำซ้ำที่ไม่ได้เพิ่มความหมาย เช่น "เขารู้สึกว่ามีอะไรบางอย่างที่ไม่ชอบมาพากล" → "มีบางอย่างผิดปกติ"
- ปรับคำบรรยายที่แปลตรงจนแข็ง ให้เป็นสำนวนไทยที่ลื่น (แต่คงภาพและอารมณ์เดิม)
- **ข้อห้ามเด็ดขาด**: ห้ามเพิ่มประโยคบรรยาย ห้ามเพิ่มรายละเอียดประสาทสัมผัส (เสียง/กลิ่น/สัมผัส) ที่ต้นฉบับไม่มี — "ไหลลื่น" ไม่ใช่ "แต่งเติม" คงปริมาณเนื้อหาเท่าต้นฉบับเสมอ

### มิติ 4: จังหวะและความลื่นไหล (Rhythm & Flow)

ทำให้อ่านเหมือนนิยายไทยที่เขียนดี ไม่ใช่บทแปล:

- สลับความยาวประโยค — ห้ามใช้ประโยคยาวเท่ากันหมดหรือสั้นเท่ากันหมด
- ฉาก action/ต่อสู้ → ประโยคสั้น กระชับ จังหวะเร็ว ตัดคำฟุ่มเฟือย
- ฉาก emotion/บรรยาย → ประโยคยาวขึ้นได้ ให้ผู้อ่านดื่มด่ำ
- ฉาก dialogue → ตัดคำนำที่ซ้ำ เช่น "เขาพูดว่า" "เธอตอบว่า" ทุกบรรทัด — ใช้ action beat แทน
- อ่านในใจ — ถ้าสะดุดตรงไหน แสดงว่าต้องแก้ตรงนั้น

## ข้อห้าม

- ห้ามเปลี่ยนเนื้อหา ข้อมูลหลัก หรือชื่อเฉพาะ
- ห้ามเพิ่มเนื้อหาใหม่ที่ต้นฉบับไม่มี
- ห้ามลดทอนอารมณ์หรือตัดฉากสำคัญ
- ห้ามใส่เชิงอรรถหรือหมายเหตุในเนื้อเรื่อง
- ห้ามเปลี่ยนศัพท์เฉพาะที่ OKF กำหนด
- ห้ามเปลี่ยนน้ำเสียงตัวละครที่ `voice-register.md` กำหนด

## อัปเดตสถานะ

หลังบันทึกฉบับเกลาแล้ว ให้อัปเดตสถานะ **ผ่านสคริปต์** (ตรวจไฟล์ edited จริงก่อนเขียน):

```powershell
powershell -File etc/set-status.ps1 -Chapter {CHAPTER_NUMBER} -Stage edited
```

หลังเกลา ให้สรุป:

- จุดที่แก้ตาม QA report
- Translation-ese ที่ลบ (ยกตัวอย่าง 2-3 จุด)
- คำ/ประโยคที่ทำให้ลื่นขึ้น (ยกตัวอย่าง 2-3 จุด)
- จุดที่ปรับจังหวะ/flow
- ยืนยันว่า **ไม่ได้เพิ่มเนื้อหา/บรรยายใหม่** ที่ต้นฉบับไม่มี
- จุดที่ยังควรให้มนุษย์ตรวจ
