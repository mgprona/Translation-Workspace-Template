# Prompt: Bootstrap OKF

สร้าง OKF จาก source ต้นฉบับก่อนเริ่มแปล — ใช้ตอนเปิดโปรเจกต์ใหม่ที่ OKF ยังว่างเปล่า

## ขั้นตอน

### 1. อ่าน Source สำรวจ

**ก่อนอื่น** ตรวจว่า source พร้อมและชื่อโฟลเดอร์ตรงมาตรฐาน:

```powershell
powershell -File etc/verify-sources.ps1
```

ถ้า `[FAIL]` (ไม่เจอ source หรือชื่อโฟลเดอร์ไม่ตรง) → **หยุด** แก้ source ก่อน (ดู SETUP.md) ห้าม bootstrap จากการเดา

อ่าน source **5 ตอนแรก** เพื่อสำรวจภาพรวม:

- `sources/eng_clean_chapter/ch001.txt` ถึง `ch005.txt`
- `sources/raw_chapter/ch001.txt` ถึง `ch005.txt` (ถ้ามี)

ถ้ามี `sources/full_text.txt` ให้ scan คร่าวๆ เพื่อจับภาพรวม (ไม่ต้องอ่านทั้งหมด แค่ดูชื่อตัวละคร/ศัพท์สำคัญที่ปรากฏบ่อย)

### 2. สกัดองค์ความรู้

จาก source ที่อ่าน ให้สกัดรายการดังนี้:

- **ตัวละคร**: ชื่อทุกตัวที่ปรากฏ พร้อมบทบาทคร่าวๆ
- **สำนัก/ฝ่าย/องค์กร**: ทุกกลุ่มที่กล่าวถึง
- **สถานที่**: ทุกสถานที่ที่กล่าวถึง
- **วิชา/ทักษะ/เทคนิค**: ทุกอย่างที่เป็นชื่อเฉพาะ
- **วัตถุ/สมบัติ/อาวุธ**: ทุกชิ้นที่มีชื่อเฉพาะ
- **ศัพท์เฉพาะ**: คำที่ต้องแปลสม่ำเสมอ (ตำแหน่ง, แนวคิด, ระบบ, หน่วยวัด, เงินตรา ฯลฯ)
- **น้ำเสียงตัวละคร**: สรรพนาม, ระดับภาษา, ลักษณะการพูด
- **ชื่อบท**: ชื่อบทอังกฤษ/เกาหลี ของตอนที่อ่าน

### 3. เสนอคำแปลไทย

สำหรับทุกรายการที่สกัดได้ ให้เสนอคำแปลไทย พร้อมเหตุผลสั้นๆ:

- ชื่อคน → ทับศัพท์ตามเสียงต้นฉบับ (ใช้ต้นฉบับดิบภาษาต้นทางเป็นหลักถ้ามี)
- ชื่อวิชา/สำนัก/ตำแหน่ง → แปลความหมายเป็นไทยถ้าทำได้, ทับศัพท์ถ้าเป็นชื่อเฉพาะ
- ศัพท์ระบบ → เลือกคำไทยที่เป็นธรรมชาติ ไม่แข็งเกินไป

ถ้าไม่มั่นใจ ให้ mark เป็น `review-needed` ใน `human-review-needed.md`

### 4. เขียนลง OKF

**อ่านไฟล์ OKF เป้าหมายก่อนเขียนทุกครั้ง** — แต่ละไฟล์มี example row ใน comment (`<!-- ... -->`) บอกรูปแบบแถวที่ถูกต้อง ใช้เป็นแนว แล้วเติมข้อมูลจริงต่อจากหัวตาราง (จะลบ comment ตัวอย่างทิ้งหรือคงไว้ก็ได้)

เติมรายการลงไฟล์ OKF ที่เกี่ยวข้อง:

| สิ่งที่พบ | ไฟล์ OKF |
|----------|---------|
| ตัวละคร | `okf/characters.md` |
| สำนัก/ฝ่าย/องค์กร | `okf/factions.md` |
| สถานที่ | `okf/places.md` |
| วิชา/ทักษะ | `okf/techniques.md` |
| วัตถุ/สมบัติ | `okf/artifacts.md` |
| ศัพท์เฉพาะ | `okf/terms.md` |
| น้ำเสียงตัวละคร | `okf/voice-register.md` |
| ชื่อบท | `okf/title-registry.md` |

### 5. อัปเดตไฟล์เสริม

- แทนค่า placeholder `{DATE}` (= วันที่วันนี้) และ `{NOVEL_NAME}` ในทุกไฟล์ที่ยังค้างอยู่ ตามตารางใน `SETUP.md`
- `okf/source-map.md` — อัปเดตจำนวน source files และ coverage
- `okf/chapter-registry.md` — ลงรายชื่อบทที่มี source พร้อมคอลัมน์ `Coverage` (เช่น `korean+english` หรือ `english-only` ตามต้นฉบับที่มีจริงของตอนนั้น)
- `okf/style-guide.md` — วิเคราะห์ประเภทนิยายแล้วตั้งค่า **Genre Profile** (Tone, ระดับภาษา, คำต้องหลีก, ระดับการบรรยาย) และเพิ่ม voice targets
- `reports/batch-plan.md` — **เสนอขอบเขต arc แรก**: จาก source ที่สำรวจ ประเมินว่าศึก/ช่วงเนื้อเรื่องแรกน่าจะจบราวตอนไหน (เป้า ~30 ตอน ยืดหยุ่น 28-34 ตัดที่รอยต่อเนื้อเรื่อง) เสนอเป็นช่วง `START-END` ให้ผู้ใช้ยืนยัน แล้วลงแถว arc 1 (Phase = A)
- `logs/translation-decisions.md` — บันทึกการตัดสินใจศัพท์ทั้งหมดที่ทำ

หลังแทน placeholder แล้ว ให้ยืนยันว่าไม่มี `{DATE}`/`{NOVEL_NAME}` ตกค้าง (ลืมง่ายเพราะมีหลายไฟล์):

```powershell
powershell -File etc/check-placeholders.ps1
```

ถ้าขึ้น `[ISSUE]` ให้ไปแทนค่าไฟล์/บรรทัดที่ระบุให้ครบก่อนไปต่อ

### 6. ลบคอมเมนต์ตัวอย่างที่ไม่ใช้แล้ว

ลบ example-row comment (`<!-- ตัวอย่างรูปแบบแถว... -->`) ในไฟล์ OKF ที่เติมข้อมูลจริงแล้ว — คอมเมนต์เหล่านี้มีไว้สอนรูปแบบตอน bootstrap เท่านั้น ถ้าทิ้งไว้จะกิน token ทุกครั้งที่อ่าน OKF (สะสมมากในเรื่องยาวหลายร้อยตอน) ส่วนไฟล์ที่ยังว่างจะคงคอมเมนต์ไว้ก็ได้

### 7. สรุปให้ผู้ใช้

หลังจบ bootstrap ให้สรุป:

- จำนวนรายการที่เพิ่มในแต่ละไฟล์ OKF
- รายการที่ยังไม่มั่นใจ (อยู่ใน `human-review-needed.md`)
- ประเภท/โทนของนวนิยาย (กำลังภายใน, แฟนตาซี, Sci-fi, ฯลฯ)
- คำแนะนำ style สำหรับนวนิยายเรื่องนี้
- **ขอบเขต arc 1 ที่เสนอ** (ช่วงตอน + เหตุผลที่ตัดตรงนั้น) ให้ผู้ใช้ยืนยัน
- พร้อมเริ่ม **Phase A ของ arc 1** ด้วย `prompts/01-translate-chapter.md` (draft+QA รายตอนจนครบ arc ก่อนเกลา)

## ข้อห้าม

- ห้ามเดาคำแปลที่ไม่มั่นใจแล้วล็อกเป็นมาตรฐาน — ใส่ `human-review-needed.md` แทน
- ห้ามข้ามตัวละครหรือศัพท์ที่เห็น แม้จะปรากฏแค่ครั้งเดียว
- ห้ามแปลโดยไม่ดู raw source ถ้ามี (ชื่อคนต้องตรวจเสียงจากต้นฉบับภาษาต้นทางเสมอ)

## หมายเหตุ

OKF ที่ได้จาก bootstrap จะ**ยังไม่สมบูรณ์** — จะค่อยๆ เติบโตผ่าน `prompts/04-update-okf.md` ขณะแปลตอนต่อๆ ไป Bootstrap เป็นแค่ฐานเริ่มต้นเพื่อให้ตอนแรกๆ ไม่แกว่ง
